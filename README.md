## 🆕 최신 업데이트 (2026-02-20)

### 📧 이메일 알림 시스템 구현 ⭐ NEW!

**공지사항이 올라오면 선택한 회원에게 자동으로 이메일이 전송됩니다!**

#### 주요 기능
- **수신자 선택 발송**: 관리자가 원하는 회원만 선택하여 이메일 발송
- **전체 발송 옵션**: "전체 선택" 버튼으로 모든 회원에게 일괄 발송
- **실시간 공지 전달**: 공지사항 제목과 내용이 그대로 이메일로 발송
- **체크박스 선택**: 직관적인 UI로 회원 선택
- **무료**: EmailJS 무료 플랜 사용 (월 200통)
- **서버 불필요**: 정적 사이트에서 바로 작동
- **자동 필터링**: 이메일이 없는 회원은 자동으로 비활성화
- **2줄 레이아웃**: "이메일" + "알림보내기" 2줄 배치로 컴팩트하고 깔끔한 디자인
- **균등한 버튼 크기**: PC 화면에서 "공지사항 등록"과 "이메일 알림보내기" 버튼이 동일한 크기로 배치
- **반응형 최적화**: 모바일에서는 버튼 크기가 축소되어 화면에 적절하게 배치

#### 사용 방법
```
1. 관리자 로그인
2. 공지사항 작성 (제목 + 내용 입력)
3. "📧 이메일로 알림 보내기" 버튼 클릭
4. 모달창에서 수신자 선택:
   - "✅ 전체 선택": 모든 회원 선택
   - "❌ 전체 해제": 모든 선택 해제
   - 개별 체크박스: 원하는 회원만 선택
5. "📧 선택한 회원에게 발송" 버튼 클릭
6. 확인 → 선택한 회원에게 자동 전송!
```

#### 문제 해결 가이드

**버튼을 클릭해도 아무 반응이 없다면?**

1. **관리자로 로그인했는지 확인**
   - 일반 회원은 이메일 발송 불가
   - 우측 상단에 "👑 관리자" 배지가 보여야 함

2. **공지사항 제목과 내용을 입력했는지 확인**
   - 제목과 내용이 모두 입력되어야 버튼 작동
   - 비어있으면 "공지사항을 먼저 작성해주세요" 알림

3. **브라우저 콘솔 확인 (F12)**
   - 버튼 클릭 시 콘솔에 다음 로그가 나타나야 함:
   ```
   [EMAIL] 📧 이메일 모달 열기 함수 호출됨!
   [EMAIL] 현재 사용자: {...}
   [EMAIL] 관리자 여부: true
   [EMAIL] ✅ 모달 열기 시작
   [EMAIL] ✅ 모달 열기 완료
   ```

4. **캐시 지우고 새로고침**
   - Ctrl + Shift + R (Windows)
   - Cmd + Shift + R (Mac)

5. **테스트 페이지로 확인**
   - https://bttprayer.net/test-email.html 접속
   - 각 테스트 버튼을 눌러 시스템 상태 확인
```

   - 로그인 후 비밀번호 변경 권장

**방법 2: API를 통한 비밀번호 업데이트 (개발자용)**

1. **회원 정보 확인**
   - 관리자 페이지에서 "회원 목록" 확인
   - 비밀번호를 설정할 회원의 ID 확인

2. **비밀번호 해시 생성**
   - 브라우저 개발자 도구 (F12) 열기
   - Console 탭에서 다음 코드 실행:
   ```javascript
   // 예: "password123"을 해시로 변환
   const password = "password123"; // 설정할 비밀번호
   const msgBuffer = new TextEncoder().encode(password);
   crypto.subtle.digest('SHA-256', msgBuffer).then(hashBuffer => {
       const hashArray = Array.from(new Uint8Array(hashBuffer));
       const hashHex = hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
       console.log('비밀번호 해시:', hashHex);
       // 이 해시 값을 복사하세요
   });
   ```

3. **API로 회원 정보 업데이트**
   - Console에서 다음 코드 실행:
   ```javascript
   // 회원 ID와 해시 값으로 업데이트
   const memberId = "회원ID"; // 1단계에서 확인한 ID
   const passwordHash = "해시값"; // 2단계에서 복사한 값
   
   fetch(`tables/members/${memberId}`, {
       method: 'PATCH',
       headers: {'Content-Type': 'application/json'},
       body: JSON.stringify({ password: passwordHash })
   }).then(r => r.json()).then(data => {
       console.log('업데이트 완료:', data);
       alert('비밀번호가 설정되었습니다!');
   });
   ```

4. **회원에게 안내**
   - 설정한 비밀번호를 회원에게 안전하게 전달
   - 회원이 로그인 후 비밀번호를 변경하도록 안내

**방법 3: 회원에게 재가입 안내**
1. 기존 회원 계정 삭제
2. 회원에게 재가입 요청
3. 회원이 직접 비밀번호 설정

**💡 보안 주의사항:**

브라우저 개발자 도구(F12)의 Console 탭을 열면 다음 정보를 확인할 수 있습니다:

1. **API 호출 상태**
   - `[API] 기도 제목 로드 시작...`
   - `[API] 기도 제목 응답 상태: 200 OK`
   - `[API] 기도 제목 데이터: {...}`

2. **재시도 메커니즘**
   - 서버 오류(500번대) 발생 시 최대 3회 자동 재시도
   - 네트워크 오류 발생 시 자동 재시도 (지수 백오프 방식)
   - `[API RETRY] 시도 2/4 - URL: tables/prayers...`
   - `[API RETRY] 서버 오류 (500). 2000ms 후 재시도...`

3. **에러 상세 정보**
   - `[API ERROR] 기도 제목 로드 실패: {...}`
   - `[ERROR] 기도 제목을 불러오는 중 오류 발생: Error: ...`

#### 일반적인 오류와 해결 방법

**422 오류 (Unprocessable Entity)**
- **원인**: 데이터 검증 실패 (잘못된 데이터 형식)
- **해결**: 
  1. 콘솔에서 상세 오류 메시지 확인
  2. 입력한 데이터 형식 확인
  3. 필수 필드가 모두 입력되었는지 확인

**500 오류 (Server Error)**
- **원인**: 서버 내부 오류
- **해결**: 
  1. 자동으로 3회 재시도됩니다
  2. 계속 실패하면 잠시 후 다시 시도
  3. 문제가 지속되면 관리자에게 문의

**네트워크 오류**
- **원인**: 인터넷 연결 문제
- **해결**: 
  1. 인터넷 연결 상태 확인
  2. 자동으로 재시도됩니다
  3. 페이지 새로고침

**CORS 오류**
- **원인**: 브라우저 보안 정책
- **해결**: 
  1. 페이지 배포 후 정상 작동합니다
  2. 로컬 테스트 시에는 무시 가능

### 관리자 기능 테스트

관리자 기능이 제대로 작동하는지 확인하려면:

1. **테스트 페이지 접속**: `test-admin.html` 페이지를 엽니다
2. **로그인 상태 확인**: 최지연 관리자로 자동 로그인되었는지 확인
3. **관리자 권한 확인**: "✅ 관리자 권한 있음" 메시지 확인
4. **API 테스트**: 버튼을 클릭하여 데이터베이스 연결 확인

### 데이터베이스 오류 시

만약 "Failed to load resource: the server responded with a status of 500" 오류가 발생하면:

1. **자동 재시도 확인**: 시스템이 자동으로 3회 재시도합니다. 콘솔에서 재시도 로그 확인
2. **프로젝트 배포 확인**: Publish 탭에서 프로젝트가 정상적으로 배포되었는지 확인
3. **데이터베이스 연결**: 백엔드 서버가 정상 작동하는지 확인
4. **브라우저 캐시 삭제**: Ctrl+Shift+Delete로 캐시와 쿠키 삭제 후 재접속
5. **로컬스토리지 초기화**: `test-admin.html` 페이지에서 "로컬스토리지 초기화" 버튼 클릭

**3. 재시도 설정 조정**
```javascript
// js/main.js의 API_RETRY_CONFIG 수정
const API_RETRY_CONFIG = {
    maxRetries: 3,      // 최대 재시도 횟수

```

**4. 오류 발생 시 확인사항**

- 오류 메시지의 status code 확인 (422, 500 등)
- 네트워크 탭에서 실제 API 요청/응답 확인
